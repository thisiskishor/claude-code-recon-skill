# Contributing

Contributions are welcome — improvements to the skill instructions, new reference seeds, bug fixes, or better trigger descriptions.

## How to Contribute

1. Fork the repo
2. Create a branch: `git checkout -b my-improvement`
3. Make your changes
4. Open a pull request with a clear description of what changed and why

## What's Worth Contributing

- **Better trigger phrases** in `SKILL.md` frontmatter — if you found a prompt that should trigger the skill but doesn't, add it to the description
- **New scalability seeds** in `references/domain-wisdom.md` — keep the same format (metaphor, what it unfolds into, violation/honored examples)
- **Improvements to `CLAUDE.md`** — if you found a behavioral rule that makes Claude more careful or useful in practice
- **Example reports** in `examples/` — real or anonymized topology reports from actual codebases

## What to Avoid

- Don't add features that only apply to one specific stack or language
- Don't add MUSTs and ALWAYSs without explaining the reasoning behind them
- Don't pad the skill with things Claude can already do without guidance
