## Executive Summary

A mid-sized Node.js/React SaaS application with a monorepo structure. Backend is Express + PostgreSQL via Supabase. Frontend is Next.js 14 with Tailwind. Auth is handled via Supabase Auth with JWT. Three background workers run via pg_cron.

**Key risks identified:**
- `pg_cron` used for HTTP fan-out to external APIs ‚Äî this is an architecture risk (see Risk Summary)
- No test coverage on `/api/billing` routes (Critical coverage gap)
- Two unused service files found ‚Äî likely dead code from a previous payment provider

## Risk Summary

| Area | Rating | Justification |
|---|---|---|
| `pg_cron` ‚Üí `pg_net` for HTTP fan-out | **High** | Database used as HTTP dispatcher ‚Äî hits background-worker ceiling under load. Violates Seed 6 ("right tool for the right job") and Seed 13 ("database stores truth, not dispatches work"). Recommend migrating to an external scheduler. |
| No tests on `/api/billing` | **Critical** | Billing routes handle money movement with zero test coverage. A regression here causes revenue impact with no automated detection. |
| Missing input validation on `PATCH /api/users` | **High** | Raw `req.body` passed to update query. No schema validation. Risk of mass assignment or injection if query construction changes. |
| Rate limiting gaps on billing | **Medium** | `/api/billing/checkout` unprotected ‚Äî vulnerable to checkout flooding. |
| Dead service files (`paddle.ts`, `chargebee.ts`) | **Low** | No runtime risk, but adds confusion and maintenance overhead. Remove. |

## Recommended Next Steps

1. **Immediate:** Add `zod` schema validation to `PATCH /api/users` and `POST /api/billing/checkout`
2. **Immediate:** Write integration tests for all billing routes
3. **Short-term:** Migrate pg_cron HTTP fan-out to an external scheduler (Inngest, Trigger.dev, or a simple cron worker)
4. **Short-term:** Delete `services/paddle.ts`, `services/chargebee.ts`, and `middleware/csrf.ts`
5. **Short-term:** Add rate limiting to billing endpoints
6. **Nice to have:** Add payload assertion tests to `tests/users.test.ts` ‚Äî status-only assertions miss regression bugs
